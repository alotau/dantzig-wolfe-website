## Problem Description

A paper mill receives orders for rolls of paper of varying widths. The mill produces large master rolls of a fixed width $W$. Each master roll must be cut into smaller rolls to fill a set of orders. The goal is to minimise the number of master rolls used (equivalently, to minimise waste) while satisfying all orders.

A **cutting pattern** is a feasible assignment of widths to a master roll:

<InlineMath formula="\sum_{i=1}^{n} w_i a_{ij} \le W, \quad a_{ij} \in \mathbb{Z}_{\ge 0}" />.

## Block-Angular LP Formulation

The cutting stock problem can be stated as:

<MathBlock
  formula={`
\\min \\quad & \\sum_{j=1}^{m} x_j \\\\
\\text{s.t.} \\quad & \\sum_{j=1}^{m} a_{ij} x_j \\ge d_i \\quad \\forall i = 1, \\ldots, n \\quad \\text{(coupling constraints)} \\\\
& x_j \\ge 0, \\quad x_j \\in \\mathbb{Z}_+ \\quad \\forall j
`}
  display={true}
/>

The LP relaxation sets <InlineMath formula="x_j \ge 0" /> (replacing integrality). Because there are exponentially many patterns <InlineMath formula="m" />, this LP cannot be solved by enumerating all columns — Dantzig-Wolfe Decomposition via **column generation** is the key technique.

<Callout type="info">
**Coupling constraints**: The demand constraints <InlineMath formula="\sum_j a_{ij} x_j \ge d_i" /> are the **coupling constraints** — they link all patterns together and are retained in the master problem.

There are no explicit sub-problem constraints separating by index in this formulation; instead, the sub-problem is the knapsack feasibility check for generating new columns.

</Callout>

## Dantzig-Wolfe Master Problem

Introduce dual variables <InlineMath formula="\pi_i \ge 0" /> for each demand constraint. The restricted master problem (RMP) maintains only a subset of columns <InlineMath formula="J \subseteq \{1, \ldots, m\}" />:

<MathBlock
  formula={`
\\min \\quad & \\sum_{j \\in J} x_j \\\\
\\text{s.t.} \\quad & \\sum_{j \\in J} a_{ij} x_j \\ge d_i \\quad \\forall i \\\\
& x_j \\ge 0 \\quad \\forall j \\in J
`}
  display={true}
/>

At each iteration, solve the RMP and obtain dual variables <InlineMath formula="\pi_i^*" />.

**Role of dual variables**: <InlineMath formula="\pi_i^*" /> is the shadow price of demand <InlineMath formula="i" /> — the marginal reduction in total rolls used if demand <InlineMath formula="i" /> were increased by one unit. These prices drive the column generation sub-problem.

## Pricing Sub-Problem (Knapsack)

Check whether any currently unlisted pattern can improve the objective. The reduced cost of a new pattern <InlineMath formula="a = (a_1, \ldots, a_n)^T" /> is:

<MathBlock formula={`\\bar{c}(a) = 1 - \\sum_{i=1}^{n} \\pi_i^* a_i`} display={true} />

We seek a pattern with **negative reduced cost** — i.e., one that reduces total rolls. This is equivalent to solving a **0-1 knapsack problem**:

<MathBlock
  formula={`
\\max \\quad & \\sum_{i=1}^{n} \\pi_i^* a_i \\\\
\\text{s.t.} \\quad & \\sum_{i=1}^{n} w_i a_i \\le W \\\\
& a_i \\in \\mathbb{Z}_{\\ge 0} \\quad \\forall i
`}
  display={true}
/>

If <InlineMath formula="\max \sum_i \pi_i^* a_i > 1" />, a new improving column exists and is added to the RMP. Otherwise, the RMP is optimal.

## Numerical Iteration (Example)

Consider <InlineMath formula="n = 2" /> order widths, <InlineMath formula="W = 10" />, <InlineMath formula="w_1 = 4" />, <InlineMath formula="w_2 = 3" />, <InlineMath formula="d_1 = 3" />, <InlineMath formula="d_2 = 5" />.

**Initial patterns**: Let <InlineMath formula="p_1 = (2, 0)^T" /> (two rolls of width 4) and <InlineMath formula="p_2 = (0, 3)^T" /> (three rolls of width 3).

**Iteration 1**:

- Solve RMP with <InlineMath formula="\{p_1, p_2\}" />.
- Obtain duals <InlineMath formula="\pi_1^* = 0.5" />, <InlineMath formula="\pi_2^* = 0.333" />.
- Knapsack: maximise <InlineMath formula="0.5 a_1 + 0.333 a_2" /> subject to <InlineMath formula="4a_1 + 3a_2 \le 10" />.
- Optimal solution: <InlineMath formula="a_1 = 1, a_2 = 2" /> giving value <InlineMath formula="0.5 + 0.667 = 1.167 > 1" />.
- Add new pattern <InlineMath formula="p_3 = (1, 2)^T" />.

**Iteration 2**:

- Re-solve RMP with <InlineMath formula="\{p_1, p_2, p_3\}" />.
- Updated duals and check knapsack again.
- If no pattern achieves reduced cost <InlineMath formula="< 0" />, LP is optimal.

<Callout type="tip">
  This iterative process — solve RMP, extract duals, solve knapsack, add column — is exactly column
  generation. In practice, only a small fraction of all possible patterns need ever be generated.
</Callout>
