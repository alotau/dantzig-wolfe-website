# Contract: Solver Web Worker Message Protocol

**File**: `src/workers/solver.worker.ts`  
**Date**: 2026-03-07

This contract defines every message type that may be sent between the main thread
(`src/lib/solver/worker-client.ts`) and the Pyodide Web Worker. All messages use the
structured clone algorithm via `postMessage`. No transferable objects are used.

---

## Main Thread â†’ Worker Messages

### `INIT`

Sent once when the Svelte island mounts. Triggers Pyodide load and package installation.
No payload.

```typescript
interface InitMessage {
  type: 'INIT';
}
```

**Worker behaviour**: Loads Pyodide from CDN, installs `micropip`, installs
`dantzig-wolfe-python` wheel, then sends `READY`.

---

### `SOLVE`

Sent when the user clicks "Solve" with a valid `ProblemInstance`.

```typescript
interface SolveMessage {
  type: 'SOLVE';
  requestId: string;          // UUID; echoed back in all response messages for this solve
  payload: ProblemInstance;   // Defined in data-model.md
}
```

**Worker behaviour**: Converts `ProblemInstance` to the Python solver's input format,
calls the `dantzig_wolfe.solve()` function, streams `ITERATION` messages per iteration,
then sends a final `RESULT` message.

---

### `CANCEL`

Sent when the user clicks "Cancel" during an active solve.

```typescript
interface CancelMessage {
  type: 'CANCEL';
  requestId: string;   // Must match the in-progress SOLVE requestId
}
```

**Worker behaviour**: Sets a cancellation flag that is checked after each iteration
boundary in the Python callback. The current iteration completes; then the worker sends
`RESULT` with `status: 'cancelled'`.

---

## Worker â†’ Main Thread Messages

### `READY`

Sent once after successful Pyodide and package initialisation.

```typescript
interface ReadyMessage {
  type: 'READY';
  pyodideVersion: string;       // e.g. "0.29.3"
  solverPackageVersion: string; // e.g. "1.2.0"
}
```

---

### `INIT_ERROR`

Sent if Pyodide fails to load or the package cannot be installed.

```typescript
interface InitErrorMessage {
  type: 'INIT_ERROR';
  error: string;   // Human-readable description
}
```

---

### `ITERATION`

Sent after each column generation iteration completes. May be sent many times per solve.

```typescript
interface IterationMessage {
  type: 'ITERATION';
  requestId: string;
  payload: SolverIteration;   // Defined in data-model.md
}
```

---

### `RESULT`

Sent once when the solve is complete (regardless of outcome).

```typescript
interface ResultMessage {
  type: 'RESULT';
  requestId: string;
  payload: SolverResult;   // Defined in data-model.md
}
```

---

## Type Union

```typescript
// Outbound from main thread
type WorkerInboundMessage = InitMessage | SolveMessage | CancelMessage;

// Outbound from worker
type WorkerOutboundMessage =
  | ReadyMessage
  | InitErrorMessage
  | IterationMessage
  | ResultMessage;
```

---

## Error Handling Rules

1. If the worker throws an unhandled exception during a solve, it MUST send a `RESULT`
   message with `status: 'error'` and `errorMessage` set before the worker thread exits.
2. The main thread MUST handle the case where the worker fails to send `READY` within
   30 seconds (`INIT_ERROR` or timeout) and display an appropriate message to the user.
3. `requestId` MUST be a globally unique string (UUID v4) generated by the main thread.
   The worker MUST echo it on every response related to that request.
